Key differences

  • TypeScript's any is different from Swift's any. TypeScript will not type-check any at all, which can be dangerous.
  • Swift has both optional types and optional arguments (optional arguments have default values).
  • TypeScript functions can't have optional arguments with default values (the default value implies optional)
  • TypeScript's null is sometimes used to represent non-existant "objects" (object, array, or function values) - but most developers use undefined instead.

func myFunction(
  arg1: String?

// Optional type and default value
func myFunction(
  arg1: String? = nil
) => String {}
      

function myFunction(
  arg1: string
) => string {}






// Default value
function myFunction(
  arg1: string = "hello"
) => string {}

// Argument with optional type
function myFunction(
  arg1: string | undefined
) => string {}






// Optional argument
function myFunction(
  arg1?: string
) => string {}

Key differences

  • Swift arguments have optional "local names"
  • Swift has both optional types and optional arguments (optional arguments have default values).
  • TypeScript functions can't have optional arguments with default values (the default value implies optional)
